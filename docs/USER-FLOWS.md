# Parcours utilisateurs

## 1. Premier lancement

### Objectif

Permettre à un parent de rendre l’application utilisable en moins de trois minutes.

### Parcours

1. Présentation courte du principe.
2. Confirmation que l’application est destinée à un usage familial privé.
3. Création du premier profil enfant :
   - prénom ou pseudonyme ;
   - tranche d’âge ;
   - niveau de lecture ;
   - avatar.
4. Choix du mode de validation par défaut.
5. Création d’un code parent.
6. Sélection de trois modèles de quêtes suggérés.
7. Arrivée dans le monde enfant avec une première quête disponible.

### Contraintes

- aucun compte ;
- aucune date de naissance complète ;
- aucun formulaire long ;
- possibilité de passer les réglages facultatifs ;
- retour arrière possible sans perdre les données saisies.

## 2. Créer une quête depuis un modèle

1. Le parent ouvre « Quêtes ».
2. Il sélectionne « Ajouter ».
3. Il choisit une catégorie ou recherche un modèle.
4. Il ouvre un modèle.
5. Il modifie éventuellement le titre, la consigne ou l’illustration.
6. Il sélectionne l’enfant concerné.
7. Il choisit le jour ou la récurrence.
8. Il confirme la validation et la récompense proposées.
9. Il enregistre.
10. Une synthèse indique quand la quête apparaîtra.

Objectif : moins de 30 secondes sans personnalisation avancée.

## 3. Créer une quête personnalisée

1. Le parent saisit un titre court.
2. Il choisit une illustration ou une catégorie.
3. Il rédige une consigne enfant.
4. Il ajoute éventuellement des étapes.
5. Il attribue la quête et sa planification.
6. Il choisit le type de validation.
7. L’application vérifie que la consigne est courte et compréhensible.
8. Le parent enregistre.

Les options avancées restent repliées par défaut.

## 4. Découvrir les quêtes du jour

1. L’enfant ouvre son profil.
2. Le monde affiche une réaction courte de la mascotte.
3. Les quêtes disponibles apparaissent sous forme de grandes cartes.
4. Les quêtes nécessaires et facultatives sont visuellement distinctes sans vocabulaire culpabilisant.
5. L’enfant touche une carte.
6. La consigne est illustrée et peut être lue à voix haute.
7. L’enfant quitte l’écran pour réaliser la quête.

## 5. Commencer et terminer une quête

1. L’enfant ouvre la quête.
2. Il peut indiquer « Je commence » ou directement la réaliser.
3. Les éventuelles étapes sont présentées une par une.
4. L’enfant touche « J’ai terminé ».
5. Selon la règle :
   - validation immédiate ;
   - demande à un adulte ;
   - validation ensemble.
6. L’application renvoie rapidement vers le monde réel si la validation attend un adulte.

## 6. Valider côté parent

1. Une demande apparaît dans le tableau de bord parent.
2. Le parent ouvre la demande.
3. Il voit la quête, l’enfant et l’heure de demande.
4. Il choisit :
   - valider ;
   - regarder ensemble ;
   - indiquer qu’une petite étape reste à faire.
5. La décision est enregistrée.
6. En cas de validation, la récompense est attribuée une seule fois.

Le bouton de refus n’utilise jamais « Échec » ou « Faux ».

## 7. Découvrir une récompense

1. Après validation, une célébration courte démarre.
2. L’objet, l’habitant ou le fragment d’histoire apparaît.
3. Une phrase explique ce qui a changé dans le monde.
4. L’enfant peut toucher « Voir dans mon monde ».
5. L’animation se termine automatiquement et ne bloque pas la navigation.

La célébration respecte le mode réduction des animations.

## 8. Journée sans quête terminée

1. L’application ne montre aucun compteur rouge.
2. Les occurrences passées restent consultables côté parent.
3. Le parent peut ignorer, reporter ou laisser l’occurrence telle quelle.
4. L’enfant retrouve le lendemain un accueil neutre ou chaleureux.
5. Aucune mascotte triste, aucun objet perdu, aucune série cassée.

## 9. Gérer plusieurs enfants

1. Le parent crée plusieurs profils.
2. Une quête peut être individuelle ou attribuée à plusieurs enfants.
3. Chaque enfant possède sa progression propre.
4. Le sélecteur de profil montre uniquement avatars et prénoms.
5. Aucune comparaison de progression n’est affichée.
6. Le parent peut consulter les activités par enfant sans classement.

## 10. Mettre une routine en pause

1. Le parent ouvre une planification.
2. Il choisit « Suspendre ».
3. Il renseigne éventuellement une date de reprise.
4. Aucune occurrence n’est créée pendant la pause.
5. Les anciennes réalisations restent intactes.

Cas d’usage : vacances, maladie, changement temporaire de rythme.

## 11. Sauvegarder et restaurer

### Export

1. Le parent ouvre les réglages.
2. Il choisit « Sauvegarder mes données ».
3. L’application précise si les photos sont incluses.
4. Le fichier est généré localement.
5. Un avertissement rappelle que le fichier contient des données familiales.

### Import

1. Le parent sélectionne un fichier.
2. L’application le vérifie sans modifier les données actuelles.
3. Un résumé du contenu est affiché.
4. Le parent confirme le remplacement.
5. Une sauvegarde automatique de l’état actuel est créée.
6. L’import est effectué dans une transaction.

## 12. Mise à jour de l’application

1. Une nouvelle version est détectée.
2. L’application termine les écritures en cours.
3. Elle propose une mise à jour à un moment non bloquant.
4. Les données sont sauvegardées avant migration.
5. La migration est exécutée et validée.
6. En cas d’échec, l’ancienne version des données reste restaurable.

## 13. Mode enfant sans lecture

Chaque action principale doit être compréhensible grâce à :

- l’illustration ;
- la position constante des boutons ;
- une icône explicite ;
- une lecture vocale ;
- une phrase très courte facultative.

Aucun écran enfant critique ne doit exiger la lecture d’un paragraphe.
